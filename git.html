Git / Git hub

GIT

GIT이란 소스코드를 효율적으로 관리하기 위해 사용 하는 버전 관리 시스템이다.
(VCS : Version Control System)

그렇다면 왜 효율적으로 관리하는 GIT을 사용해야 할까?

개발자들이 협업하거나 파일들을 유지/보수 할때, 특히 코드를 수정했지만 수정하기 전으로 되돌려야 할 때 등
버전별 소스코드를 저장 및 관리가 가능한 GIT이 필요하다.

GIT의 개념은 타임머신을 가지고 있는 요리사로 생각하면 간단하다.
간장을 3T 넣었는데 짜다면? 다시 시간을 돌려서 2T를 넣으면되듯,
GIT을 통해 수정 이전으로 돌아가는 것은 더욱 완벽한 개발을 하는데 매우 용이하다.

1. 버전관리

git을 사용하지 않은 버전관리를 생각해보자.
1. 코드를 짠다.
2. 저장한다. 저장명 : 코드1
3. 코드를 수정한다.
4. 저장한다. 저장명 : 코드2
5. 코드를 수정한다
6. 저장한다. 저장명 : 코드3
이하 무한 반복
하지만 Git을 통해 버전을 관리한다면?

그렇다면 어떻게 버전관리를 할까?

COMMIT

- 저장하고자 하는 전체 내용이 사진을 찍듯 박제가 됨.
이후에도 의미있는 변경사항이 있을 때 마다 commit 을 통해 변경사항만 저장할 수 있음
>commit을 여러번해도 용량을 몇배로 차지하지 않음

PULL

PUSH

BRANCH

한개의 스토리에 두가지 결말이 있다면?
프로젝트에서도 결정하기가 애매한 부분에서 Branch를 통해 작업 내용을 분기시킬 수 있음
주가 되는 코드는 메인 브렌치에서, 시도해볼만한 코드는 이외의 브렌치에서 작업을 이어가고
후에 실제로 적용을 원한다면 메인 브렌치에서 합칠 수 있음
즉, 버전별로 매번 따로 복사&저장할 필요가 없음!

2. 백업

다른 컴퓨터에 "복제" 하는 것 > 내 컴퓨터에서 유실되었다고 하더라도 언제든 다시 불러낸다.
내가 만들어낸 것을 다른 어떤 컴퓨터에 저장할 것인가?
그것을 도와주는 것이 "git hub"

여기서 잠깐!
git과 git hub의 차이점
git에서는 자신의 로컬 소스코드를 관리(add,commit,merge) 할 수 있고
git hub는 내가 로컬에서 git으로 관리하는 자료들을 다른 사람과 공유하거나 '백업' 하는 웹사이트
[정리하자면]
git : local내에서 소스코드를 관리하는 것 (Git에는 다양한 종류가 있음)
github : local에서 관리한 소스코드를 업로드하고 공유할 수 있는 공간 (Git의 종류 중 하나로 가장 많이 사용됨)

3. 협업

규모가 있는 프로젝트는 혼자 코딩을 하지 않을 것!
다른 구성원이 작업한 내용을 다운받고, 내가 작업한 것을 전송하기도 함
이 과정들이 체계적으로 기록되고 관리되기 때문에 코드를 수정한 작업자를 찾아내거나
과거로 코드를 되돌려 문제를 해결할 수 있음
